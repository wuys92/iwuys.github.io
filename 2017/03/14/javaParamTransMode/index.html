java参数传递方式?

  • java

如果没有接触过这个问题，那么对这个问题的答案可能是java参数既有值传递也有引用传递。然而后来有一次，我们经理给我们出的一个考题就考到了这个，毫无意外的答错。然后今天在网上看到有人提了一个这样的问题，想起了这个问题就先记下来吧。。。。

###值传递和引用传递的含义及特点

含义：在方法调用时，传递的参数是按值的拷贝传递。
特点：传递的是值的拷贝，也就是说传递后就互不相关了


含义：在方法调用时，传递的参数是按引用进行传递，
     其实传递的引用的地址，变量所对应的内存空间的地址。
特点：传递的是值的引用，即传递前和传递后都指向同一个引用
    （也就是同一个内存空间）

So..talk is cheap,show me the code.

上面这段代码试图去改变testList的引用地址，如果参数是引用传递，那么最后打印的内容应该是“talk is cheap show me the code”，但是最后内容是“i dont wanna talk”，说明方法里面改变对象引用地址是不可以的。

ps.各位可以自己试一下传递基本数据类型参数和改变参数对象的参数值

>
其实运行完例子之后我们就可以得出结论了

  • 一个方法不能修改一个基本数据类型的参数
  • 一个方法可以改变一个对象（数组）参数的状态
  • 一个方法不能让对象参数（数组）引用一个新的对象。

那java的参数传递为啥是这样？

先给传递的变量和参数变量命个名，方便理解。
main(String[] args){paramX; testParamTrans(paramX);}
testParamTrans(paramY){...}

对于基本数据类型的参数的传递

  1. 把值拷贝一份给参数，即上面代码里面的paramX值拷贝了一份给paramY（其实这个时候就已经注定paramX和paramY缘分已尽了）。
  2. testParamTrans方法压栈，执行testParamTrans方法，哪怕改变paramY变量的值，对paramX也是毫无影响。
  3. testParamTrans弹栈，paramY变量不再使用。

对于对象参数的传递

其实和基本数据类型的参数的传递方式差不多，只不过传递的是对象的引用内存地址值，然后paramY复制了一份地址值。因此paramY也是指向paramX这个对象的，可以改变这个对象内部的属性；但是paramY改变引用内存地址值，对paramX还是没有没有任何影响的。
